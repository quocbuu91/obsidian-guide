Bước 1 · Khởi đầu
Tải & Cài đặt Obsidian

Obsidian là công cụ ghi chú cục bộ (local-first) — dữ liệu lưu trên máy bạn, không phụ thuộc đám mây. Hoàn toàn miễn phí cho cá nhân.

  • ✓
    Truy cập obsidian.md → nhấn Download → chọn phiên bản phù hợp (Windows / macOS / Linux)
  • ✓
    Cài đặt ứng dụng và mở lần đầu tiên
  • ✓
    Chọn Create new vault khi được hỏi
  • ✓
    Đặt tên vault: startup-[tên-dự-án] và chọn thư mục lưu trữ
💡
Vault là gì? Vault = kho tài liệu. Mỗi vault là một thư mục trên máy tính. Bạn có thể có nhiều vault cho nhiều dự án khác nhau.
⚠️
Lưu ý: Đặt vault trong Google Drive / OneDrive nếu muốn tự động sao lưu lên đám mây — không cần phải trả phí Obsidian Sync.

Bước 2 · Nền tảng
Hiểu giao diện & Markdown cơ bản

Obsidian dùng ngôn ngữ Markdown — cực kỳ đơn giản, bạn sẽ thành thạo trong 10 phút.

📝
Editor (giữa)
Nơi bạn viết ghi chú. Nhấn Ctrl+E để chuyển giữa chế độ Edit và Preview.
🗂️
File Explorer (trái)
Cây thư mục dự án. Kéo thả file để sắp xếp, click phải để tạo folder mới.
🔗
Graph View (Ctrl+G)
Xem bản đồ liên kết giữa các ghi chú — điểm độc đáo nhất của Obsidian.
🔍
Quick Switch (Ctrl+O)
Mở nhanh bất kỳ file nào bằng tên — cực kỳ hữu ích khi vault lớn.

Markdown cần nhớ ngay:

# Tiêu đề lớn (H1) ## Tiêu đề vừa (H2) ### Tiêu đề nhỏ (H3) **in đậm** *in nghiêng* ~~gạch ngang~~ - Danh sách không thứ tự - [ ] Checkbox chưa làm - [x] Checkbox đã xong [[Tên ghi chú]] ← liên kết nội bộ (quan trọng nhất!) #tag-nhãn ← gắn thẻ để tìm kiếm
🎯
Thực hành ngay: Tạo file mới (Ctrl+N), thử gõ các cú pháp trên, nhấn Ctrl+E để xem kết quả render.

Bước 4 · Liên kết
Dùng Internal Links & Tags để kết nối ý tưởng

Đây là sức mạnh thực sự của Obsidian — mọi ý tưởng đều có thể liên kết với nhau, tạo thành mạng lưới kiến thức sống động.

Cách dùng Internal Link [[...]]:

# Problem Statement Khách hàng của chúng ta (xem [[User Interviews/User-01]]) phản ánh rằng việc quản lý đơn hàng thủ công tốn quá nhiều thời gian. ## Giải pháp tiềm năng - Xem thêm ý tưởng trong [[Solution Ideas]] - Đối thủ đã làm gì? → [[Competitor Map]] ## Tags liên quan #pain-point #ưu-tiên-cao #đã-validate

Hệ thống Tags nên dùng cho Startup:

#trạng-thái
#idea → mới nảy ra
#in-progress → đang làm
#đã-validate → đã kiểm chứng
#đã-loại → không dùng
#lĩnh-vực
#product
#marketing
#finance
#user-research
🗺️
Graph View magic: Sau khi tạo nhiều liên kết, nhấn Ctrl+G để thấy bản đồ kết nối của toàn bộ dự án — cực kỳ ấn tượng khi thuyết trình với mentor!

Bước 5 · Templates
Tạo Templates cho mọi loại tài liệu

Templates giúp mỗi lần tạo tài liệu mới đều đúng format, tiết kiệm thời gian và đồng nhất cho cả team.

Bật Core Plugin Templates:

  • ✓
    Vào Settings → Core plugins → Templates → bật ON
  • ✓
    Tạo folder _Templates/ trong vault
  • ✓
    Trong Settings → Templates → chỉ đường dẫn đến folder _Templates

Template: Biên bản họp (Meeting Note)

--- date: {{date}} attendees: type: meeting --- # Meeting: {{title}} ## 🎯 Mục tiêu cuộc họp - ## 💬 Nội dung thảo luận - ## ✅ Action Items - [ ] Người thực hiện: | Deadline: - [ ] Người thực hiện: | Deadline: ## 📎 Tài liệu liên quan - [[]] #meeting #{{date:YYYY-MM}}

Template: Phỏng vấn người dùng

--- date: {{date}} interviewee: segment: --- # User Interview — {{date}} ## 👤 Thông tin người dùng - Tuổi / Nghề nghiệp: - Segment: ## 🔥 Pain Points phát hiện - ## 💡 Insights quan trọng - ## 📊 Quote đáng nhớ > "..." #user-research #interview
⚡
Dùng template: Tạo file mới → Ctrl+P → gõ "Insert template" → chọn template. Hoặc dùng hotkey tùy chỉnh trong Settings.

Bước 6 · Plugins
Cài Community Plugins thiết yếu

Obsidian có marketplace plugin mạnh mẽ. Đây là 5 plugin không thể thiếu cho startup — tất cả đều miễn phí.

📅 Dataview Quan trọng nhất
Query dữ liệu từ ghi chú như SQL. VD: liệt kê tất cả Action Items chưa xong trong mọi cuộc họp.
📋 Kanban
Tạo bảng Kanban (To Do / Doing / Done) ngay trong Obsidian — quản lý sprint của team.
📆 Calendar + Daily Notes
Ghi chú hằng ngày (daily log), liên kết với lịch — theo dõi tiến độ startup từng ngày.
🎨 Excalidraw
Vẽ sơ đồ, flowchart, wireframe ngay trong Obsidian. Cực hữu ích khi brainstorm cùng team.
🤖 Copilot (AI)
Tích hợp AI vào Obsidian — tóm tắt, phân tích ghi chú, brainstorm ý tưởng mới.
🔧
Cách cài: Settings → Community plugins → Turn off Safe mode → Browse → tìm tên plugin → Install → Enable.

Ví dụ Dataview query thực tế:

```dataview TASK WHERE !completed AND contains(tags, "#meeting") SORT file.mtime DESC ``` ← Liệt kê TẤT CẢ action items chưa xong từ mọi cuộc họp

Bước 7 · Workflow
Thiết lập quy trình làm việc cho cả Team

Obsidian cá nhân thì tốt, nhưng dùng chung với team mới thực sự mạnh. Đây là workflow đề xuất cho startup 2–5 người.

Quy trình hằng ngày (Daily Workflow):

Sáng
Mở Daily Note (Ctrl+Shift+D) → ghi 3 mục tiêu hôm nay → link đến task liên quan
Trưa
Cập nhật Kanban board → tick task xong → ghi lại blockers
Tối
Quick retrospective 5 phút trong Daily Note → "Học được gì hôm nay?" → capture ý tưởng mới vào Inbox

Chia sẻ Vault với Team — 3 cách:

☁️
Google Drive / OneDrive
Miễn phí, đơn giản nhất. Để vault trong shared folder. Chú ý: không nên mở cùng lúc một file.
🐙
GitHub (Git plugin)
Mạnh nhất — version control đầy đủ, xem history thay đổi, resolve conflict. Phù hợp team technical.

🚀
Pro tip cuối: Tạo file 🏠 Home.md làm dashboard — embed Dataview queries để xem tất cả tasks đang mở, meetings gần nhất, và ý tưởng chưa validate — ngay khi mở vault mỗi sáng.
